# 导航组件 · 使用与配置手册

> 依据 [`官网组件清单.md`](./官网组件清单.md)（客户建议拆分）编写，覆盖**导航 3 个组件**：header、页脚、地区/语言切换 Tab。
> 面向对象：Universal Editor 内容作者 + 前端开发。

## ⚠️ 与首页组件的关键差异：导航是「片段/文档」驱动，不是 UE 字段

首页组件在 UE 里靠「容器 + 子项 + Dialog 字段」配置；**header 和 footer 不同**——它们的内容来自作者维护的两份**独立文档（fragment）**：

| 组件 | 内容来源文档 | 覆盖元数据 | 加载方式 |
| --- | --- | --- | --- |
| header | 默认 `/nav`（相对内容根 `/content/demo-site/language-master/en`） | 页面 `nav` 元数据可覆盖路径 | `loadFragment(navPath)` |
| 页脚 footer | 默认 `/footer` | 页面 `footer` 元数据可覆盖路径 | `loadFragment(footerPath)` |

作者**编辑 `/nav`、`/footer` 这两篇文档的正文结构**（标题、列表、链接、图片），header/footer 的 JS 再把这些语义结构解析成导航/页脚 UI。不需要、也没有独立的 UE Dialog 字段。

> 内容环境：`author-p80707-e1685574.adobeaemcloud.com`，内容根 `/content/demo-site/language-master/en`（见 `Universal-editor-demo/AGENTS.md`，非用户明示不切换）。

## 1. header（全局顶部导航）

**优先级 P0** · 对应 block：`header`（`/nav` 文档驱动）

![header PC](./assets/导航__header__1.png)

### 用途
全站顶部导航。承载品牌 Logo、主导航入口、车型 mega 面板、右侧工具区（立即定购 / 登录 / 语言）。

### `/nav` 文档结构（作者编辑对象）
header.js 把 `/nav` 文档按 **3 个 section** 顺序解析：

| section | 角色 | 作者放什么 |
| --- | --- | --- |
| 第 1 段 | **Brand 品牌** | 一个带图片的链接：Logo 图 + 首页链接（`brandLink`/`brandImg`/`brandImgAlt`） |
| 第 2 段 | **Primary 主导航** | 一个链接列表 = 顶级导航项（Vehicles / Official Center / Service / Media Center / About Us…）。某项下**嵌套子列表**即成为该项的下拉/mega 面板 |
| 第 3 段 | **Tools/Language 工具区** | 语言/工具入口链接（`toolsLink`/`toolsLabel`，默认文案 `Language`） |

### 车型 mega 面板（Vehicles 下拉）
- 在第 2 段「车型」这一项下**嵌套子列表**，子项被 `extractDropdownEntry` 解析、`groupDropdownEntries` 按分类分组，渲染成车型卡面板。
- 卡片 = 车型名（title）+ 定位文案（subtitle）+ 车图（media）。
- 分组示例（截图）：**Battery Electric Vehicle**（Li i8 / Li i8 / Li MEGA）、**REEV Vehicle**（Li L6 / L7 / L8 / L9）。分组名来自子列表里的分组标题。
- 导航项 hover 有浅蓝色热区触发面板。

![车型 mega 面板](./assets/导航__header__2.png)

### 主题（透明 / 白色）
- 由页面 `header-theme` 元数据 + 滚动状态控制。
- **客户最终规则**：**仅首页导航半透明**（下滑变为白导航）；**其他页面默认白导航**。
  （截图中「所有车型/技术/充电/零售/服务页也半透明」一条已被划掉，以「仅首页半透明」为准。）

### 作者操作
1. 打开 `/nav` 文档。
2. 第 1 段放 Logo 图 + 首页链接。
3. 第 2 段编辑主导航链接列表；需要车型面板就在「车型」项下加嵌套子列表（分组标题 + 各车型链接/图/定位文案）。
4. 第 3 段配置语言/工具入口。
5. 各页需要透明头时设 `header-theme` 元数据（默认白；首页设半透明）。

### ⚠️ 已知差距
- 右侧「立即定购 / 登录」按钮在不同市场截图中有/无（Global 版只有地球图标，无登录/定购）——需确认是否按 market 控制。

## 2. 页脚 footer

**优先级 P0** · 对应 block：`footer`（`/footer` 文档驱动）

![footer](./assets/导航__页脚__1.png)

### 用途
全站底部：多列链接导航 + 法务信息 + 社交媒体 + 回到顶部。

### `/footer` 文档结构（作者编辑对象）
footer.js 把 `/footer` 文档按 **2 个 section** 解析：

| section | 角色 | 作者放什么 |
| --- | --- | --- |
| 第 1 段 | **栏目导航** | 若干「标题（h2–h6）+ 紧跟的链接列表 `<ul>`」→ 每个标题成为一列。示例：Vehicles（Li i6/i8/MEGA/L6/L7/L8/L9）、Official Center、Service、Media Center、About Us（About Us/Join Us/Investor Relations） |
| 第 2 段 | **底栏** | 段落里的法务文案 + 政策链接 + 社交链接。示例：© Li Auto Inc. 2026、京ICP备19003172号-9、Cookie Policy、Privacy Policy；社交：Instagram / LinkedIn / Facebook / YouTube |

- 栏目 = 「标题 + 列表」组合，加/减列就是加/减标题块。
- 底栏社交图标来自第 2 段里的链接（`querySelectorAll('a')`），按平台渲染图标。
- 内置**回到顶部**按钮：滚动超过 `300px` 出现，点击平滑滚回顶部。

### 作者操作
1. 打开 `/footer` 文档。
2. 第 1 段用「标题 + 无序列表」写各栏目；调整栏目即增删标题块。
3. 第 2 段写版权/ICP、政策链接、社交链接（社交自动转图标）。

## 3. 地区/语言切换 Tab

**优先级 P2** · 归属：**内置于 `header`**（非独立 block）

![地区语言弹窗](./assets/导航__地区_语言切换Tab__1.png)

### 用途
header 右上角地球图标 🌐 打开「Choose Your Country and Language / Select a region and language」弹窗，切换市场与语言。

### ⚠️ 当前实现方式：硬编码，非作者可配
- 市场/语言列表在 **`blocks/header/header.js` 的 `LOCALE_MARKETS` 常量里硬编码**，不由 `/nav` 文档或 UE Dialog 驱动。
- 弹窗标题常量 `LOCALE_DIALOG_TITLE = 'Select a region and language'`。
- **当前代码内置市场**：Global（English）、China（简体中文 → lixiang.com）、Қазақстан（Қазақ тілі / русский язык）、Oʻzbekiston（O'zbek tili / русский язык）。

### ⚠️ 与设计稿的差距
- 设计截图弹窗为 **5 个市场**，比代码多一个 **البحرين（Bahrain / العربية 阿拉伯语）**；标题文案设计为 "Choose Your Country and Language"（代码为 "Select a region and language"）。
- 若要新增市场/语言或改标题，**需改 `header.js` 代码**（`LOCALE_MARKETS` + 标题常量），当前无作者侧配置入口。是否改为可配置需与客户/前端确认。

### 作者/开发操作
- 作者：无可配项（当前）。
- 开发：在 `blocks/header/header.js` 的 `LOCALE_MARKETS` 增删市场/语言、对齐弹窗标题；改后本地 `aem up` 验证弹窗与跳转。
